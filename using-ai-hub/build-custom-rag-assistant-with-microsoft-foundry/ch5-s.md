# Challenge 05: Generate Knowledge-Grounded Responses and Enable Telemetry

## Introduction

With the retrieval pipeline working, the next step is to combine **retrieved documents** with a **generative AI model** to produce meaningful responses. This is the core concept of **Retrieval-Augmented Generation (RAG)**.

When a user asks a question, the system first retrieves the most relevant documents from the search index. These documents are then passed to the language model as context, allowing it to generate responses that are **grounded in real knowledge instead of hallucinating answers**.

In addition to generating responses, modern AI systems must also support **observability and monitoring**. Telemetry logging helps developers track how the system behaves in production, including queries, responses, and performance metrics.

In this challenge, you will test the **RAG chat application**, generate responses grounded in retrieved knowledge, and enable **telemetry monitoring using Application Insights**.

## Challenge Objectives

- Generate AI responses using retrieved knowledge  
- Test the RAG chat application  
- Create an **Application Insights** resource  
- Enable telemetry logging for monitoring the RAG pipeline  

## Steps to Complete

### Generate Responses Using the RAG Pipeline

1. In **Visual Studio Code**, open the file:

   `chat_with_products.py`

1. This script performs the following tasks:

   - Retrieves relevant product documents from the search index  
   - Sends retrieved documents to the language model  
   - Generates a grounded response using the GPT model  

1. No code modification is required.

1. In the **VS Code terminal**, ensure you are inside the directory:

   `rag/custom-rag-app`

1. Run the following command to test the RAG assistant:

   ```
   python chat_with_products.py --query "I need a new tent for 4 people, what would you recommend?"
   ```

1. Review the response generated in the terminal.

1. The response should include recommendations based on the retrieved product data.

### Create an Application Insights Resource

1. Navigate back to the **Microsoft Foundry portal**. Configure **Tracing** with new Application Insights with name `Applicationinsight`.

### Install Telemetry Monitoring Package

1. Return to **Visual Studio Code**.

1. In the **terminal**, install the telemetry monitoring package by running:

   ```
   pip install azure-monitor-opentelemetry
   ```

1. Wait for the installation to complete.

### Enable Telemetry Logging

1. First, sign in to Azure (required for telemetry authentication):

   ```
   az login
   ```

1. Sign in using the lab credentials provided as **Work or School** account.

1. Run the RAG chat application again with telemetry enabled:

   ```
   python chat_with_products.py --query "I need a new tent for 4 people, what would you recommend?" --enable-telemetry
   ```

1. Navigate back to Tracing page.

   > **Note:** Telemetry data may take **5–10 minutes** to appear.

### Filter Telemetry Results

1. In the **Tracing dashboard**, click **Filter**.

1. Select **+ Add filter**.

1. Configure the filter with the following values:

   - **Field:** Success  
   - **Operator:** Equal to  
   - **Value:** True  

1. Click **Apply**.

1. The dashboard will now display only successful requests generated by the RAG application.

## Success Criteria

- RAG chat application successfully generates grounded responses  
- Application Insights resource created successfully  
- Telemetry monitoring package installed  
- Query telemetry appears in the Foundry tracing dashboard
